# วิธีใช้งาน (templates/namo_forbidden_archive)

ไฟล์ตัวอย่างในโฟลเดอร์นี้ออกแบบมาเพื่อให้คุณคัดลอกไปวางใน repository ของโปรเจค `NaMo Forbidden Archive` โดยไม่แก้ไขเนื้อหาภายในโปรเจคหลัก

- `context7.json`: กำหนดการสแกนของ Context7 — ตัวอย่างนี้ตั้งให้ *exclude* ไฟล์และโฟลเดอร์ที่มีเนื้อหา sensitive/NSFW (เช่น `llms.txt`) เพื่อป้องกันการ index โดยไม่ตั้งใจ
- `.env.example`: ตัวอย่างตัวแปรสภาพแวดล้อม (placeholders) — อย่า commit ค่า API keys จริงลงใน repo สาธารณะ

ขั้นตอนแนะนำ:

1. คัดลอก `context7.json` ไปวางที่ root ของ repo `NaMo_Forbidden_Archive` (หรือสร้าง PR ถ้าคุณไม่ได้เป็นเจ้าของ)
2. ถ้าต้องการให้ Context7 รีสแกน: เปิดหน้า Dashboard ของ Context7 -> ไปที่โปรเจค -> กด `Manage` -> `Refresh` / `Re-scan`
3. ถ้าคุณต้องการเก็บไฟล์นี้สำหรับใช้ภายในเท่านั้น ให้อย่าทำให้ repo เป็น public และอย่า commit `.env` ที่มีคีย์จริง

ถ้าต้องการให้ผมช่วย:

- ผมสามารถสร้าง PR ตัวอย่างใน repo (ต้องให้สิทธิ์หรือคุณยอมให้ผม fork และส่ง PR)
- ผมสามารถช่วยเขียนสคริปต์ตรวจสอบว่าไฟล์ `llms.txt` ถูกยกเว้นหลังรีสแกน โดยใช้ `@upstash/context7-sdk` (คุณต้องรันด้วย `CONTEXT7_API_KEY`)

แจ้งให้ผมรู้ว่าต้องการให้ผมช่วยอันไหนต่อได้เลยครับ
